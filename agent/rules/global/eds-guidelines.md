# Global EDS XWalk Guidelines

These rules apply to all EDS blocks/pages generated by this agent.

## Workflow (mandatory)

- **Step 1**: Generate backend XWalk JSON first (block-level JSON + models).
- **Step 2**: Require **user-provided Universal Editor HTML**. Do not invent HTML.
- **Step 3**: Generate FE JS/CSS using **index-based** extraction matching the UE HTML.

## Block implementation rules

- **Index-based only**: use `block.children[n]`, `row.children[n]` for structure. Do not use `data-*` selectors for structure or selection.
- **One JS/CSS per block**: exactly `blocks/<block>/<block>.js` and `blocks/<block>/<block>.css`.
- **Parent/child in one folder**: if a pattern uses child items, keep a single JS/CSS pair.
- **Defensive coding**: use optional chaining and null checks to handle missing optional rows/cells.

## Authoring model rules

- Field order is the structure contract. Document it in the JS `decorate()` JSDoc.
- Field `name` should be **camelCase** (avoid underscores).
